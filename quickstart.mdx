This guide walks you through setting up the Sentry client and reporting your first error.

## Installation

First, install the library:

```bash
go get github.com/altipla-consulting/sentry
```

## Setup

<Steps>
  <Step title="Get your Sentry DSN">
    Log in to your Sentry account and copy your project's DSN from **Settings > Projects > [Your Project] > Client Keys (DSN)**.

    The DSN looks like:
    ```
    https://examplePublicKey@o0.ingest.sentry.io/0
    ```
  </Step>

  <Step title="Set environment variables">
    Store your Sentry DSN in an environment variable:

    ```bash
    export SENTRY_DSN="https://your-dsn@sentry.io/project-id"
    export VERSION="1.0.0"  # Optional: for release tracking
    ```

    <Note>
    The `VERSION` environment variable is automatically used by the client to tag errors with your application version.
    </Note>
  </Step>

  <Step title="Initialize the client">
    Create a Sentry client in your application:

    ```go main.go
    package main

    import (
        "context"
        "fmt"
        "os"
        "time"

        "github.com/altipla-consulting/sentry"
    )

    func main() {
        // Initialize the Sentry client
        client := sentry.NewClient(os.Getenv("SENTRY_DSN"))
        
        // Ensure all events are sent before exiting
        defer client.Flush(5 * time.Second)

        // Your application code here
        if err := run(); err != nil {
            client.Report(context.Background(), err)
        }
    }

    func run() error {
        return fmt.Errorf("something went wrong")
    }
    ```

    <Info>
    If the DSN is empty, `NewClient` returns `nil`, and all operations become no-ops. This makes it safe to use in environments where Sentry is not configured.
    </Info>
  </Step>

  <Step title="Run your application">
    Run your application:

    ```bash
    go run main.go
    ```

    Check your Sentry dashboard - you should see the error reported!
  </Step>
</Steps>

## Report errors with context

For better error tracking, use the context helpers to attach metadata:

```go
package main

import (
    "context"
    "fmt"
    "os"
    "time"

    "github.com/altipla-consulting/sentry"
    sentrygo "github.com/getsentry/sentry-go"
)

func main() {
    client := sentry.NewClient(os.Getenv("SENTRY_DSN"))
    defer client.Flush(5 * time.Second)

    // Create a context with Sentry scope
    ctx := sentry.WithContext(context.Background())

    // Add tags for filtering
    sentry.Tag(ctx, "environment", "production")
    sentry.Tag(ctx, "user_id", "12345")

    // Add breadcrumbs to track flow
    sentry.LogBreadcrumb(ctx, sentrygo.LevelInfo, "app", "Application started")
    sentry.LogBreadcrumb(ctx, sentrygo.LevelInfo, "database", "Connected to database")

    // Report an error with all context
    err := processOrder(ctx)
    if err != nil {
        client.Report(ctx, err)
    }
}

func processOrder(ctx context.Context) error {
    sentry.LogBreadcrumb(ctx, sentry.LevelInfo, "order", "Processing order")
    return fmt.Errorf("failed to process order: %w", fmt.Errorf("payment declined"))
}
```

## Track HTTP requests

For web applications, use `WithRequest` to automatically capture HTTP request details:

```go
package main

import (
    "fmt"
    "net/http"
    "os"
    "time"

    "github.com/altipla-consulting/sentry"
)

var sentryClient *sentry.Client

func main() {
    sentryClient = sentry.NewClient(os.Getenv("SENTRY_DSN"))
    defer sentryClient.Flush(5 * time.Second)

    http.HandleFunc("/", handler)
    http.ListenAndServe(":8080", nil)
}

func handler(w http.ResponseWriter, r *http.Request) {
    // Attach Sentry scope to the request
    r = sentry.WithRequest(r)

    // Add custom tags
    sentry.Tag(r.Context(), "endpoint", "/")

    // Handle the request
    if err := processRequest(r); err != nil {
        sentryClient.Report(r.Context(), err)
        http.Error(w, "Internal Server Error", http.StatusInternalServerError)
        return
    }

    w.Write([]byte("OK"))
}

func processRequest(r *http.Request) error {
    return fmt.Errorf("database connection failed")
}
```

## Recover from panics

Use `ReportPanics` with defer to catch and report panics:

```go
package main

import (
    "context"
    "os"
    "time"

    "github.com/altipla-consulting/sentry"
)

func main() {
    client := sentry.NewClient(os.Getenv("SENTRY_DSN"))
    defer client.Flush(5 * time.Second)

    ctx := sentry.WithContext(context.Background())
    defer client.ReportPanics(ctx)

    // This panic will be caught and reported to Sentry
    panic("unexpected error occurred")
}
```

<Warning>
Always use `defer client.Flush()` before your application exits to ensure all events are sent to Sentry. The recommended timeout is 5 seconds.
</Warning>

## Complete example

Here's a complete example showing error unwrapping and stack traces:

```go
package main

import (
    "context"
    "fmt"
    "os"
    "time"

    "github.com/altipla-consulting/sentry"
)

func main() {
    client := sentry.NewClient(os.Getenv("SENTRY_DSN"))
    defer client.Flush(5 * time.Second)

    ctx := sentry.WithContext(context.Background())
    sentry.Tag(ctx, "module", "order-service")

    // Simulate a complex error chain
    err := layer1("order-123")
    if err != nil {
        client.Report(ctx, err)
    }
}

func layer1(orderID string) error {
    if err := layer2(orderID); err != nil {
        return fmt.Errorf("layer1 failed for order %s: %w", orderID, err)
    }
    return nil
}

func layer2(orderID string) error {
    if err := layer3(); err != nil {
        return fmt.Errorf("layer2 processing: %w", err)
    }
    return nil
}

func layer3() error {
    return fmt.Errorf("database connection timeout")
}
```

This example demonstrates:

- Error wrapping with `fmt.Errorf` and `%w`
- The library automatically unwraps errors to find the root cause
- Full stack traces are captured at each layer
- Tags help filter errors in Sentry

## Available log levels

When logging breadcrumbs, you can use these severity levels:

```go
sentry.LevelDebug   // Debug information
sentry.LevelInfo    // Informational messages
sentry.LevelWarning // Warning messages
sentry.LevelError   // Error messages
sentry.LevelFatal   // Fatal errors (used for panics)
```
